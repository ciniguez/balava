3. Técnicas de Investigación de Usuario

A continuación, analizaremos las técnicas más empleadas en la industria y la academia.

A. Entrevistas Semiestructuradas

  • Descripción: Conversaciones individuales donde el investigador guía la charla mediante un guion previo, pero permite la exploración de temas emergentes.
  • Aplicabilidad: Se usa en la fase de descubrimiento. El artefacto resultante suele ser un Mapa de Empatía o la definición de User Personas.
  • Ejemplo: En el desarrollo de un sistema de gestión académica para la EPN, se entrevistan a docentes para entender cómo organizan sus calificaciones fuera del sistema actual. Se descubre que muchos usan hojas de cálculo manuales por la falta de flexibilidad del software vigente.

B. Observación Contextual (Contextual Inquiry)

  • Descripción: El investigador observa al usuario mientras este realiza sus tareas en su entorno real, interviniendo mínimamente para pedir aclaraciones ("Maestro y Aprendiz").
  • Aplicabilidad: Ideal para entender procesos complejos y flujos de trabajo. El artefacto obtenido es un Diagrama de Flujo de Tareas o un Service Blueprint.
  • Ejemplo: Para optimizar una aplicación de inventarios en una bodega, el investigador acompaña al bodeguero durante su jornada. Se observa que el usuario no puede usar ambas manos para escribir porque carga paquetes, lo que sugiere que la interfaz debe permitir comandos de voz o escaneo rápido.

C. Encuestas (Surveys)

  • Descripción: Cuestionarios estructurados enviados a una muestra representativa de usuarios para obtener datos cuantitativos.
  • Aplicabilidad: Se usa para validar hipótesis a gran escala o medir la satisfacción. El artefacto es un Informe Estadístico de tendencias y preferencias.
  • Ejemplo: Tras lanzar un módulo de inscripciones web, se envía una encuesta a 500 estudiantes para medir el nivel de satisfacción (CSAT) y el tiempo percibido de respuesta, permitiendo identificar si el rendimiento del servidor es una frustración generalizada.

D. Safari de Usuario (User Safari)

  • Descripción: El investigador se pone en los zapatos del usuario y experimenta el servicio o producto de principio a fin, documentando puntos de fricción y emociones.
  • Aplicabilidad: Se usa en la fase de Empatía. El artefacto obtenido es un Customer Journey Map (Mapa de viaje del cliente) basado en la experiencia propia.
  • Ejemplo: Para mejorar el sistema de matriculación de la facultad, los docentes se registran como si fueran alumnos nuevos, descubriendo que el proceso de subir documentos PDF es confuso y genera errores de validación que no se explican en la interfaz.

E. Talleres de Co-creación (Workshops)

  • Descripción: Sesiones colaborativas donde usuarios, diseñadores y desarrolladores trabajan juntos para idear soluciones mediante dinámicas grupales.
  • Aplicabilidad: Se usa en la fase de Ideación. Los artefactos resultantes son Prototipos de baja fidelidad (paper prototyping) o Storyboards.
  • Ejemplo: En el diseño de una nueva App para bienestar estudiantil, se organiza un workshop con representantes de la asociación de alumnos. Mediante dibujos y tarjetas, ellos proponen que la función de "chat de apoyo" sea anónima para aumentar la confianza.

F. Cliente Misterioso (Mystery Shopper)

  • Descripción: Un investigador actúa como un cliente común para evaluar la calidad del servicio, el cumplimiento de protocolos y la atención al cliente sin que el personal lo sepa.
  • Aplicabilidad: Útil para la Investigación Evaluativa de servicios híbridos (digital-físico). El artefacto es una Lista de Verificación (Checklist) o informe de cumplimiento.
  • Ejemplo: Para evaluar el soporte técnico de un software de gestión, un investigador llama fingiendo haber olvidado su contraseña. Evalúa cuánto tiempo tardan en atenderlo, si verifican su identidad y la amabilidad del trato recibido.

G. Benchmarking (Análisis Comparativo)

  • Descripción: Estudio exhaustivo de productos competidores o soluciones similares para identificar mejores prácticas, estándares de la industria y oportunidades de diferenciación.
  • Aplicabilidad: Se realiza en la Planificación/Definición. El artefacto es una Matriz de Análisis Competitivo o tabla de funcionalidades.
  • Ejemplo: Antes de programar un nuevo IDE (Entorno de Desarrollo) educativo para Java, se analizan herramientas como BlueJ, Eclipse y VS Code para determinar qué atajos de teclado son estándar y qué características de depuración visual les faltan a los actuales.

H. Grupos Focales (Focus Groups)

  • Descripción: Una técnica de recolección de datos que consiste en una entrevista grupal dirigida por un moderador, donde un grupo pequeño de usuarios (usualmente de 6 a 10 personas) discute sus opiniones, actitudes y creencias sobre un producto o concepto específico.
  • Aplicabilidad: Se utiliza principalmente en la fase de Ideación o Validación de Concepto. El artefacto que se obtiene es un Informe de Percepciones y Actitudes, que ayuda a identificar patrones de pensamiento grupal.
  • Ejemplo: Para el rediseño de un portal de servicios estudiantiles, se reúne a un grupo de estudiantes de diferentes facultades. El moderador lanza la pregunta: "¿Qué es lo primero que buscan al entrar al portal?". La discusión revela que, aunque la institución prioriza las noticias, los estudiantes solo quieren ver su horario y promedio actual en la pantalla principal.
