Singleton

El patrón de diseño singleton (instancia única) está diseñado para restringir la creación de objetos pertenecientes a una clase o el valor de un tipo a un único objeto.

Su intención consiste en garantizar que una clase sólo tenga una instancia y proporcionar un punto de acceso global a ella.

El patrón singleton se implementa creando en nuestra clase un método que crea una instancia del objeto sólo si todavía no existe alguna. Para asegurar que la clase no puede ser instanciada nuevamente se regula el alcance del constructor (con atributos como protegido o privado).

Ejercicio: Gestor de Impresoras

  • Motivación
    • En una aplicación se necesita que exista un solo Gestor de Impresoras, accesible desde muchas clases.
  • Propósito
    • Se garantiza que una clase sólo tenga una instancia y se proporciona un acceso desde cualquier otra clase.
  • Descripción del problema:
    • En una aplicación se disponen de dos impresoras: PrinterA y PrinterB, pero sólo una de ellas puede estar activa. Ambas ofrecen el método “print(String msg)”, que imprime un mensaje.
    • Se debe crear la clase Printer, abstracta. Se pretende que esta clase implemente el patrón Singleton para obtener la impresora activa. La impresora concreta se inyecta mediante principio de “Inversión de control”, a través del método “set”.
  • Que se debe hacer?
    • Diseñar con UML e implementar las clases Printer, PrinterA y PrinterB
    • Se aporta la clase PrinterMain y una aplicación para realizar la presentación de la ejecución io.5.3.jar
      • IO.in.select(Obejct[] objetos, String mensaje): dado un array de objetos, presenta una lista al usuario con los objetos (se apoya en el método toString de cada objeto) y devuelve la referencia del objeto elegida por el usuario.

Nota:

  • io.5.3.jar se encuentra en el directorio de Recursos/Patrones/ del aula virtual.
  • PrinterMain se encuentra en el diretorio de Recursos/Patrones/Singleton
